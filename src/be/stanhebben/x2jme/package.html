X2JME is written as a way to directly convert DirectX mesh files to binary jME documents. It can be used as library, as command-line tool or as AssetLoader in a jME project.

Currently, not all features are supported. The supported templates are Frame, FrameTransformMatrix, Material, Mesh, MeshMaterialList, MeshNormals, MeshTextureCoords and TextureFilename. This is enough to support all regular models, but not animations.

Custom templates are not supported, and all template definitions and unsupported template instances will be skipped.

Command - line usage

When used as command-line tool, X2JME can be used to convert files or folders. When a folder is converted, all .x files in the folder and its subfolders will be converted.

To use X2JME as command - line tool, use the java -jar x2jme.jar <arguments> command. The supported arguments are:

  • -adir <directory>: sets the assets directory (default: assets)
  • -file <file>: sets the source file or directory to convert, refers to a file or directory inside the assets directory. (default: whole assets directory)
  • -dtex <file>: sets the default texture. Refers to a file inside the assets directory. (default: none)
  • -help, -h: shows command-line help

Example: java -jar xjme.jar -adir myassets -file models -dtex dummy.bmp

In this example, the assets directory is myassets, all .x files in the myassets/models directory will be converted, and the default texture is myassets/dummy.bmp

If you notice any bugs, please let me know (x2jme.sourceforge.net). If you want any additional features, you can send me a mail too. If you request an additional feature, make sure you supply a set of DirectX files to test that feature.

Library usage

When used as library, X2JME can be used to dynamically read and convert DirectX files. In that case, XFile is the class you want to look at.

A DirectX file can be read easily by constructing an XFile instance and calling toNode() on it.

AssetLoader usage

Register XImporter as loader for .x files.

Internal operation

When a .x file is read, its first 16 bytes (the header) are processed. Then, a binary or text token stream is constructed and parsed by an LL parser.

To keep the X2JME logic simple, templates are hard-coded. Each template has its own template class. Template classes are in the be.stanhebben.x2jme.template package. Templates create data object instances when instantiated. Data item classes reside in the be.stanhebben.x2jme.data package. Template definitions and unsupported templates are skipped silently.

Defining new templates

To add new templates, a template and data class has to be written and the XTemplate class must be modified to support the new template.
